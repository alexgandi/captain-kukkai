## 6. Word Book scrolling fires tile actions — every scroll speaks a random word
*impatto 4/5 · sforzo 2/5 · touch-ergonomics · file: src/scenes/WordBookScene.js*

## Note di design (sintesi)

VISUAL LANGUAGE — 'Thai candy toybox'. One coherent system, enforced through three single sources of truth so it can never drift: (1) TYPE — Baloo 2 (chubby rounded Latin) + Mali (Thai children's reading font) injected through the existing TextStyle monkey-patch in main.js, so all 21 scenes upgrade in one place; Georgia stays only on the diploma, which is deliberately 'formal'. (2) SURFACES — UiKit is the only place that draws chrome: makeButton's candy recipe (detached soft shadow, warm cream gradient 0xfffdf6→0xfff3d9, 3px gold #ffd166 border, glossy top highlight, 70ms press squash, click sfx) gets extended to makePanel and makeIconButton, then every card, pill and emoji button in the game consumes it. Never hand-draw a rounded rect in a scene again. (3) LIGHT — one reusable radial-vignette texture, tinted per theme (volcano ember glow, ice cool wash, night deep vignette so fireflies pop, jungle nearly none), gives every scene 'stage lighting' for zero per-frame cost. Character art moves to a 4x-nominal pipeline (setDisplaySize keeps the 40x60 arcade body identical) so the AI art is finally crisp; LINEAR_MIPMAP_LINEAR then helps instead of doing nothing. HUD consolidates into one translucent glass pill (dark 0x1c1030 at 0.55, drawn hearts with damage pop, mango + word counters) — this is also what store screenshots will show. Below-the-line polish that didn't make the 25 but is worth queueing: MapScene premium pass (spline path, parchment card, pulsing next-stop beacon — its dynamic-width fix DID make item 14), HUD glass-pill redesign as a follow-on to item 13, per-frame micro-opts (getBounds scratch rects in projectile loops GameScene.js:1785/1846, Player.js:117 fxTween handle instead of tweens.isTweening, resolution:1 for frequently-updated counters), 256x256 medallion canvas or dropping mipmapFilter, and dynamic-importing post-game scenes to shrink the 1.69MB bundle.

MOBILE CAVEATS — the game is now dynamic-width (975-1100 x 450 on phones, min-clamped 800): never hardcode 800-era x positions; anchor to SAFE.left/SAFE.right for edges, W/2±offset for centered compositions, and W-relative fractions for playfields (the Market lanes bug is the canonical failure). Anything SHARED (diploma, poster) must snapshot a fixed centered 800x450 via snapshotArea — the live canvas aspect is not the artifact aspect. SAFE insets are ~55-68 game px on notched iPhones and are currently baked at create(): scenes must either use them everywhere (only 3 do today) or re-layout after Android's first-tap fullscreen changes the viewport (restart the stateless MenuScene).
